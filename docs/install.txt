Installation (Ubuntu 22.04 + Docker + Caddy + HTTPS)

Diese Anleitung beschreibt die Installation des Market-Maker Projekts auf einem Ubuntu 22.04 VPS mit Caddy als Reverse Proxy und HTTPS (Let’s Encrypt).

Voraussetzungen
	•	Ubuntu 22.04 LTS VPS
	•	Root oder sudo Zugriff
	•	Domain/Subdomains zeigen auf den VPS:
	•	panel.example.com → VPS IP
	•	api.panel.example.com → VPS IP
	•	Ports offen:
	•	80/tcp
	•	443/tcp
	•	optional 22/tcp (SSH)



Alternative: Installationsskript nutzen

Auf dem Server:

curl -O https://raw.githubusercontent.com/Mario1480/Market-Maker/main/scripts/install_vps.sh
chmod +x install_vps.sh
sudo ./install_vps.sh


⸻

1) System vorbereiten

sudo apt update && sudo apt upgrade -y
sudo apt install -y curl ca-certificates gnupg unzip git

Firewall (empfohlen):

sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 443
sudo ufw --force enable
sudo ufw status


⸻

2) Docker + Docker Compose installieren

curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER

Neue Shell öffnen (oder logout/login), dann prüfen:

docker version
docker compose version


⸻

3) Projekt nach /opt/market-maker deployen

sudo mkdir -p /opt/market-maker
sudo chown -R $USER:$USER /opt/market-maker
cd /opt/market-maker

Repo klonen (Beispiel):

git clone <REPO_URL> .


⸻

4) Caddy installieren & HTTPS aktivieren

Hinweis: Diese Anleitung nutzt das offizielle Caddy apt-Repo + systemd.

Repo + Paket installieren:

sudo bash ./scripts/install_caddy_apt.sh

Caddyfile erstellen:

sudo nano /etc/caddy/Caddyfile

Inhalt:

panel.example.com {
  reverse_proxy 127.0.0.1:3000
}

api.panel.example.com {
  reverse_proxy 127.0.0.1:8080
}

Validieren + aktivieren:

sudo caddy fmt --overwrite /etc/caddy/Caddyfile
sudo caddy validate --config /etc/caddy/Caddyfile --adapter caddyfile
sudo systemctl enable --now caddy
sudo systemctl reload caddy

HTTPS testen:

curl -I https://panel.example.com
curl -i https://api.panel.example.com/health

Beide sollten 200 liefern.

Logs (wenn Zertifikat nicht klappt):

sudo journalctl -u caddy -n 120 --no-pager


⸻

5) Environment (.env.prod) erstellen

Im Projektroot:

cd /opt/market-maker
nano .env.prod

Beispiel .env.prod (Production/HTTPS):

NODE_ENV=production

# ===== Database =====
DATABASE_URL=postgresql://mm:mm@postgres:5432/marketmaker

# ===== Admin Seed =====
ADMIN_EMAIL=admin@uliquid.vip
ADMIN_PASSWORD=CHANGE_ME
ADMIN_WORKSPACE_NAME=Main

# ===== URLs =====
# Browser → API
NEXT_PUBLIC_API_URL=https://api.panel.example.com
# Container → API (intern)
API_BASE_URL=http://api:8080

# ===== Cookies / Auth =====
COOKIE_DOMAIN=.example.com
COOKIE_SECURE=true

# ===== CORS =====
CORS_ORIGINS=https://panel.example.com,http://localhost:3000

# ===== Exchange =====
BITMART_BASE_URL=https://api-cloud.bitmart.com

# ===== SMTP / Invites =====
SMTP_HOST=smtp.hostinger.com
SMTP_PORT=465
SMTP_USER=no-reply@uliquid.vip
SMTP_PASS=CHANGE_ME
SMTP_FROM="uLiquid <no-reply@uliquid.vip>"
SMTP_SECURE=true
INVITE_BASE_URL=https://panel.example.com

# ===== Billing / Subscription (CCPayments) =====
CCPAY_APP_ID=
CCPAY_APP_SECRET=
CCPAY_BASE_URL=https://ccpayment.com
CCPAY_PRICE_FIAT_ID=1033
WEB_BASE_URL=https://panel.example.com
BILLING_PRO_MONTHLY_PRICE_CENTS=2900
BILLING_PRO_MONTHLY_AI_TOKENS=1000000
BILLING_AI_TOPUP_PRICE_CENTS=900
BILLING_AI_TOPUP_TOKENS=250000

Hinweis: Abo, Verlängerung und AI-Topups werden im UI unter Settings → Subscription verwaltet.
Hinweis: Billing-Feature-Flags (Checkout/Webhook/AI-Token-Billing) werden im Admin Backend unter /admin/billing verwaltet.

Optional: Falls Tools noch .env erwarten, kannst du .env.prod nach .env kopieren:
cp .env.prod .env


⸻

6) docker-compose.prod.yml prüfen (wichtig!)

Wichtig: In docker-compose.prod.yml gibt es keine dev‑Mounts und keine hardcoded API URLs.

Für Development nutzt du weiterhin docker-compose.dev.yml.

⸻

7) Container starten (Production)

cd /opt/market-maker
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build

Status prüfen:

docker compose --env-file .env.prod -f docker-compose.prod.yml ps

Logs:

docker compose --env-file .env.prod -f docker-compose.prod.yml logs -f --tail=200 api
docker compose --env-file .env.prod -f docker-compose.prod.yml logs -f --tail=200 web
docker compose --env-file .env.prod -f docker-compose.prod.yml logs -f --tail=200 runner


⸻

8) Zugriff testen

Web:
	•	https://panel.example.com

API Health:
	•	https://api.panel.example.com/health

Login:
	•	Admin User wird beim ersten Start automatisch angelegt (Seed), wenn keine Users existieren
	•	Login mit ADMIN_EMAIL / ADMIN_PASSWORD

⸻

9) Häufige Fehler & Fixes

A) Login schlägt fehl / NetworkError

Ursache: falsche API URL im Browser (z.B. localhost:8080).

Fix:
	•	NEXT_PUBLIC_API_URL in .env korrekt setzen
	•	Web neu bauen:

docker compose --env-file .env.prod -f docker-compose.prod.yml down
docker compose --env-file .env.prod -f docker-compose.prod.yml build --no-cache web
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d

B) Runner crasht mit Missing env: BITMART_BASE_URL

Fix: .env ergänzen:

BITMART_BASE_URL=https://api-cloud.bitmart.com

Dann:

docker compose --env-file .env.prod -f docker-compose.prod.yml up -d runner

C) HTTPS “Connection refused”

Fix:
	•	prüfen ob Caddy läuft: systemctl is-active caddy
	•	Caddy Logs: sudo journalctl -u caddy -n 120 --no-pager
	•	prüfen ob Ports lauschen: sudo ss -ltnp | egrep ':80|:443'

⸻

10) Updates deployen

cd /opt/market-maker
git pull
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build
